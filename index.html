This Python script was developed to significantly reduce the manual effort involved in processing and summarizing results from multiple SIDRA INTERSECTION files. In large projects, we often analyze dozens of junctions, each with multiple scenarios (e.g., existing, future, with improvements). Manually extracting key performance indicators (KPIs) like delays, queues, and Level of Service (LOS) from each SIDRA output file and compiling them into a summary table was extremely time-consuming and prone to errors.

Features:

  • Batch Processing: Handles multiple SIDRA output files simultaneously.

  • Error Handling: Includes robust error checking for malformed SIDRA outputs.

The script leverages Python's `os` module for file system navigation, `pandas` for data structuring and manipulation, and `openpyxl` for writing to Excel files. It demonstrates my ability to apply programming skills to solve real-world engineering workflow inefficiencies.
